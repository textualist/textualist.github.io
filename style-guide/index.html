01 Philosophy

Design principles

The design concept is to pair the feel of a well-set printed page with the utility of the terminal.

Readability is the central principle. Properly weighted fonts (400 for body text), generous line-heights, optical sizing, and considered hierarchy.

Typography is central. Source Serif 4 carries editorial content; Source Sans 3 handles infrastructure; DM Mono marks the brand.

The screen should feel like a page. The palette is drawn from parchment, ink, and a single red accent.

Restraint over decoration. One accent color, one grain texture, one signature ornament: the asterisk. No red lines, no colored rules, no decorative dashes.

The brand is utilitarian. DM Mono sets the brand name plainly, like a colophon or a terminal prompt. The asterisk is the single mark of identity.

Dark mode should feel like reading by lamplight.

Stylistic models include the London Review of Books, the Los Angeles Review of Books, and the Times Literary Supplement.

02 Brand

Brand marks

The textualist brand uses DM Mono for all marks. Display treatments share the same font, size, and tracking. The asterisk in display marks always carries a red stroke with 25% red tint fill, distinguishing it from the outlined body text. A shared nav mark (text*) appears across all contexts.

Used in sticky headers, footers, and compact contexts. DM Mono at 1rem, muted color, normal tracking. The asterisk is always --red.

Usage rules

  • The nav mark is always solid (never outlined). The asterisk is always --red.
  • All display marks use DM Mono 400. No other weights.
  • Display size is 3.5rem, tracking is 0.04em.
  • The display asterisk always uses a red stroke with 25% red tint fill.
  • In dark mode, the body stroke uses --muted instead of --ink; the asterisk stroke remains --red.

04 Typography

Type system

Three fonts, each with a clear role. Source Serif 4 handles all editorial content: body, titles, taglines. Source Sans 3 handles infrastructure: navigation, labels, tags, metadata. DM Mono serves double duty as the brand typeface and the code font. The two Source fonts share proportional DNA from Adobe's superfamily; DM Mono provides the utilitarian counterpoint.

FontRoleSource
var(--serif)All editorial content: display, titles, body, taglinesSource Serif 4 (Google Fonts), variable, optical sizing
var(--sans)Navigation, labels, tags, metadata, captionsSource Sans 3 (Google Fonts), matched companion
var(--mono)Brand marks, code blocks, technical valuesDM Mono (Google Fonts)

Weight philosophy

Weight 400 is the workhorse. Body copy, titles, card titles, and all brand marks use 400 for confident readability. Source Serif 4's optical sizing and variable weight axis allow it to remain elegant at this weight across all sizes. Heavy weights (600+) are reserved for special emphasis only. The system avoids decorative lightness: 300 is not used.

05 Texture

Paper grain

An SVG noise texture applied as a fixed overlay gives the page a tactile, printed-paper quality. In light mode it uses multiply blending at 30% opacity; in dark mode it switches to soft-light at 15% to avoid muddying the darker surface.

body::before {
  content: '';
  position: fixed;
  inset: 0;
  z-index: 9999;
  pointer-events: none;
  opacity: var(--grain-opacity);     /* 0.3 light · 0.15 dark */
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256'
    xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='n'%3E
    %3CfeTurbulence type='fractalNoise' baseFrequency='0.85'
    numOctaves='4' stitchTiles='stitch'/%3E%3C/filter%3E
    %3Crect width='100%25' height='100%25' filter='url(%23n)'
    opacity='1'/%3E%3C/svg%3E");
  background-size: 200px 200px;
  mix-blend-mode: var(--grain-blend); /* multiply light · soft-light dark */
}

08 Projects

Project branding

Each textualist project carries two forms of its name: a full name and a compact abbreviation. Both derive from the parent brand's typographic system: the full name uses the editorial serif, the abbreviation echoes the text* nav mark. Together they create a family of sub-brands that are visually distinct yet unmistakably unified under the textualist identity.

Full project names

Full names are set in Source Serif 4, 400 italic, in --red. This is the same treatment used for project titles in listings and headings. The 400 weight matches body text, keeping names present but not heavy-handed.

  font-style: italic;

  text-decoration: none;
}

11 Motion

Policy

Textualist prohibits animation and transition effects. No entrance animations, no hover transitions, no staggered reveals. Elements appear immediately. Hover states change color instantly without easing or duration.

This is a deliberate design choice rooted in the principle of restraint. Motion draws attention to the interface rather than the content. A well-set page does not need to announce itself.

Hover behavior

All interactive elements change to --red on hover. No other visual change occurs. No underlines, no shadows, no background shifts.

/* Standard hover pattern */
.nav-link:hover { color: var(--red); }
.footer-link:hover { color: var(--red); }
.card-title:hover { color: var(--red); }
